Docker로 난 무얼 하고 싶었나??

환경에 제약 없이 개발 DB를 구성하고 싶었다.
맥북에서 공부한 내용을 실습하면서 개발하고 개발한 소스 Github에 올리고 집에서는 Github에서 소스를 받아서 하면됬다.
근데, DB는?? 내가 저장한 데이터 DB 컨텍스트 정보는 환경이 바뀌면 매번 설치하고 데이터 싱크 맞추기 위해 쿼리 만들고 스크립트 돌려야 하나? 게다가 집은 Window PC이다… 개발하기도 전에 환경 구성하는 것 때문에 지친다…

그러다 문득 Docker! Docker로 이미지 만들고 Docker Hub 에 올려 놓으면 그 짜증나는 환경 구성이 해결 되지 않을까??
Docker를 왜 쓸까? 했는데 나에게 Docker가 필요해진 경우가 생긴 것이다.

Docker 기본 명령어

우선 Docker를 다루면서 자주 사용하는 기본 명령어를 정리하였습니다.
이미지를 만들고 그 이미지를 run 하여 컨테이너를 만들고 하는 작업은 반드시 하는 작업이기 때문에 정리합니다.

이미지 보기

  • docker images

이미지 실행

  • docker run imageID

이미지 삭제

  • docker rmi imageID

컨테이너 보기

  • docker ps
  • docker ps -a

컨테이너 실행

  • docker start containerID

컨테이너 중지

  • docker stop containerID

컨테이너 삭제

  • docker rm continerID

MySql 셋팅

이미지 다운로드

  • docker pull mysql:5.7

MySql 컨테이너 생성 및 실행

  • docker run -d -p 3306:3306 -e MYSQL_ROOT_PASSWORD=1111 --name ryu mysql:5.7

MySql 컨테이너 bash 쉘 접속

  • docker exec -i -t jayden-mysql bash

Docker Hub 관련 명령어

내가 작업한 MySql 컨테이너를 Docker Hub에 반영한다.

  • docker commmit container_name image_name

Docker Hub 접속

  • docker login

docker user id 환경 변수 지정

  • $ export DOCKER_ID_USER=“nicewoong”

Docker Image 태그

  • docker tag ubuntu-cpp-driver $DOCKER_ID_USER/ubuntu-cpp-driver

Docker Hub에 반영

  • docker push $DOCKER_ID_USER/ubuntu-cpp-driver
